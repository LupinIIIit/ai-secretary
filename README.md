# AI Secretary

**Piattaforma modulare per assistente telefonico e omnicanale basato su intelligenza
artificiale.**
Gestione chiamate, agenda, bridge CRM/ERP e automazioni integrate con backend Python e UX reattiva.

---

## 🧭 Stato

> **In fase di analisi** – definizione architettura, componenti e infrastruttura DevOps.

---

## 📂 Struttura prevista

- `docs/` – documentazione architetturale e tecnica
- `infra/` – configurazioni Docker, DB e pipeline
- `src/` – codice sorgente (backend + orchestratori AI)
- `ui/` – interfaccia utente desktop (Electron/React o simili)
- `scripts/` – utility e setup ambienti

---

## 🛣️ Roadmap

| Fase  | Obiettivo                               | Stato |
| ----- | --------------------------------------- | ----- |
| MVP   | Setup ambiente + core AI orchestrator   | 🔜    |
| Alpha | Integrazione chiamate + UX di base      | ⏳    |
| Beta  | Feature avanzate + automazioni complete | ⏳    |

---

## ⚙️ Requisiti di sviluppo

- macOS / Linux
- Python ≥ 3.13
- Node.js ≥ 20 (solo per UI)
- Docker + Docker Compose
- PostgreSQL ≥ 17

---

## ⚖️ Note di compliance

> Le interazioni vocali devono dichiarare chiaramente che l’interlocutore è un’intelligenza
> artificiale,
> in conformità con le normative UE sulla trasparenza AI.
